STRICT RULES:
1. Output ONLY the DOT code — no markdown, no explanation, no code fences.
2. The output must be a valid Graphviz `graph` (undirected) that viz.js can render.
3. Use the following ER diagram conventions:

NODE SHAPES:
- Regular Entity:        shape=rectangle (box)
- Weak Entity:           shape=rectangle, peripheries=2  (double rectangle)
- Regular Attribute:     shape=ellipse
- Primary Key Attr:      shape=ellipse, style="filled,underline" (use fontname with underline trick — add underscore prefix to label like "_AttrName" and note it as PK)
- Multivalued Attr:      shape=ellipse, peripheries=2
- Derived Attribute:     shape=ellipse, style=dashed
- Relationship:          shape=diamond
- Identifying Relation:  shape=diamond, peripheries=2 (for weak entity relationships)

CARDINALITY LABELS:
- Label edges between entities and relationships with cardinality: "1", "N", or "M"

LAYOUT:
- Use `rankdir=TB` for top-to-bottom layout
- Group each entity with its attributes using subgraphs for clarity

STYLE ATTRIBUTES:
- Graph: `bgcolor="transparent"`, `fontname="Arial"`, `pad="0.5"`, `splines=true`, `overlap=false`
- Entity nodes: `shape=box`, `style=filled`, `fillcolor="#DDEEFF"`, `color="#1565C0"`, `fontname="Arial"`, `fontsize=13`, `fontcolor="#0D47A1"`
- Weak entity nodes: `shape=box`, `style=filled`, `fillcolor="#DDEEFF"`, `color="#1565C0"`, `peripheries=2`

- Multivalued attribute: `shape=ellipse`, `style=filled`, `fillcolor="#FFF9C4"`, `color="#F9A825"`, `peripheries=2`
- Derived attribute: `shape=ellipse`, `style=dashed`, `fillcolor="#FFF9C4"`, `color="#F9A825"`
- Relationship nodes: `shape=diamond`, `style=filled`, `fillcolor="#E8F5E9"`, `color="#2E7D32"`, `fontname="Arial"`, `fontsize=12`

REFERENCE EXAMPLE — University ER Diagram (Student enrolls in Course, taught by Professor):

graph ER {
    bgcolor="transparent";
    fontname="Arial";
    overlap=false;
    splines=true;

    // Entities
    Student [label="Student", shape=box, style=filled, fillcolor="#DDEEFF", color="#1565C0", fontname="Arial", fontsize=13];
    Course  [label="Course",  shape=box, style=filled, fillcolor="#DDEEFF", color="#1565C0", fontname="Arial", fontsize=13];
    Professor [label="Professor", shape=box, style=filled, fillcolor="#DDEEFF", color="#1565C0", fontname="Arial", fontsize=13];

    // Student Attributes
    StudentID [label=<<u>StudentID</u>>, shape=ellipse, style=filled, fillcolor="#FFF9C4", color="#F9A825", fontsize=11];
    SName     [label="Name",      shape=ellipse, style=filled, fillcolor="#FFF9C4", color="#F9A825", fontsize=11];
    SAge      [label="Age",       shape=ellipse, style=dashed, fillcolor="#FFF9C4", color="#F9A825", fontsize=11];
    SPhone    [label="Phone",     shape=ellipse, style=filled, fillcolor="#FFF9C4", color="#F9A825", fontsize=11, peripheries=2];

    // Course Attributes
    CourseID  [label=<<u>CourseID</u>>, shape=ellipse, style=filled, fillcolor="#FFF9C4", color="#F9A825", fontsize=11];
    CTitle    [label="Title",     shape=ellipse, style=filled, fillcolor="#FFF9C4", color="#F9A825", fontsize=11];
    CCredits  [label="Credits",   shape=ellipse, style=filled, fillcolor="#FFF9C4", color="#F9A825", fontsize=11];

    // Professor Attributes
    ProfID    [label=<<u>ProfID</u>>,  shape=ellipse, style=filled, fillcolor="#FFF9C4", color="#F9A825", fontsize=11];
    PName     [label="Name",      shape=ellipse, style=filled, fillcolor="#FFF9C4", color="#F9A825", fontsize=11];

    // Relationships
    Enrolls [label="Enrolls", shape=diamond, style=filled, fillcolor="#E8F5E9", color="#2E7D32", fontsize=12];
    Teaches [label="Teaches", shape=diamond, style=filled, fillcolor="#E8F5E9", color="#2E7D32", fontsize=12];

    // Entity — Attribute connections
    Student -- StudentID;
    Student -- SName;
    Student -- SAge;
    Student -- SPhone;
    Course  -- CourseID;
    Course  -- CTitle;
    Course  -- CCredits;
    Professor -- ProfID;
    Professor -- PName;

    // Entity — Relationship connections with cardinality
    Student  -- Enrolls [label="M"];
    Course   -- Enrolls [label="N"];
    Professor -- Teaches [label="1"];
    Course   -- Teaches [label="N"];
}

- Use `graph` (undirected) NOT `digraph` for ER diagrams — all connections use `--` not `->`.
- Primary key attributes MUST use HTML label syntax: `label=<<u>AttrName</u>>` for underline.
- Every entity must connect to ALL its attributes.
- Every entity must connect to its relationships with cardinality labels.
- Multivalued attributes use `peripheries=2`.
- Derived attributes use `style=dashed`.
- Weak entities use `peripheries=2` on the box.
- Identifying relationships use `peripheries=2` on the diamond.
- Do NOT wrap the output in markdown code fences or any other formatting.
