=== DOT CODE RULES ===

Use the following Relational Schema conventions:

NODE SHAPES (TABLES):
- Every table is a single node with `shape=none` or `shape=plaintext`.
- Use an HTML-like table label for each entity. The table should have a header row for the entity name and subsequent rows for attributes.
- Use `port` attributes on rows where foreign keys connect to primary keys.

Example Table Label:
TableName [label=<
<table border="0" cellborder="1" cellspacing="0" cellpadding="4">
  <tr><td bgcolor="#1565C0"><font color="white"><b>TableName</b></font></td></tr>
  <tr><td port="pk_id" align="left">id (PK)</td></tr>
  <tr><td align="left">name</td></tr>
  <tr><td port="fk_other" align="left">other_id (FK)</td></tr>
</table>>];

RELATIONSHIP EDGES:
- Use directed edges between ports with explicit connections or simple labeled edges.
- Use `dir=both` or `dir=none` depending on connections.
- Emphasize cardinality using edge labels (e.g. `taillabel="1"`, `headlabel="N"`).
- Connect the exact row ports (e.g., `Table1:id -> Table2:fk_id`).

LAYOUT & STYLE:
- Graph: `rankdir=LR`, `bgcolor="transparent"`, `fontname="Arial"`, `splines=true`, `overlap=false`, `nodesep=0.5`, `ranksep=1.5`
- Table Header Colors: Use `#e3f2fd` (light blue) with `#1976d2` text for primary tables, `#d4edda` (light green) with `#28a745` text for association/transaction tables, `#ffcccc` (light red) with `#d32f2f` text for lookup tables, etc.
- Edges: `color="#555555"`, `fontname="Arial"`, `fontsize=10`

REFERENCE EXAMPLE — E-commerce Schema DOT:

digraph ER {

      <tr><td bgcolor="#e3f2fd"><font color="#1976d2"><b>Customers</b></font></td></tr>
      <tr><td port="id" align="left">customer_id (PK)</td></tr>
      <tr><td align="left">name</td></tr>
      <tr><td align="left">email</td></tr>
    </table>>];

    Orders [label=<
    <table border="0" cellborder="1" cellspacing="0" cellpadding="4">
      <tr><td bgcolor="#d4edda"><font color="#28a745"><b>Orders</b></font></td></tr>
      <tr><td port="order_id" align="left">order_id (PK)</td></tr>
      <tr><td port="customer_id" align="left">customer_id (FK)</td></tr>
      <tr><td align="left">order_date</td></tr>
      <tr><td align="left">total</td></tr>
    </table>>];

    // Relationship with cardinality
    Customers:id -> Orders:customer_id [taillabel="1", headlabel="N", labeldistance=2];
}

=== SQL RULES ===

1. Each array item must be one complete executable `CREATE TABLE ...;` statement.
2. End every SQL statement with a semicolon.
3. Use MySQL syntax by default. If a different dialect is specified in the input, use that dialect.
4. Extract every entity as a table.
5. Include all attributes with suitable SQL data types (INT, BIGINT, VARCHAR, TEXT, BOOLEAN, DATE, TIMESTAMP, DECIMAL, etc.).
6. Identify and define primary keys.
7. Add `AUTO_INCREMENT` (or dialect equivalent) when appropriate for surrogate keys.
8. Add NOT NULL, UNIQUE, DEFAULT, CHECK, and other constraints when implied.
9. Identify all relationships and add correct foreign key constraints.
10. For one-to-many, place FK on the many-side table.
11. For many-to-many, create a junction table with foreign keys to both parent tables.
12. Handle self-referencing relationships (e.g., manager_id -> employees.id) correctly.
13. Keep naming consistent and SQL syntax valid.
14. Order the CREATE TABLE statements so that referenced tables come before referencing tables (parent tables first).
