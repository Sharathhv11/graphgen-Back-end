── CLASS DIAGRAM ──
- Use `shape=record` for all class nodes.
- Format class labels as: `{ClassName|attribute1\lattribute2\l|method1()\lmethod2()\l}`
- Use `\l` for left-aligned line breaks inside record labels.
- Relationships:
  • Inheritance/extends: `edge [arrowhead=empty]` (hollow triangle arrow)
  • Implements: `edge [arrowhead=empty, style=dashed]`
  • Association: `edge [arrowhead=open]`
  • Composition: `edge [arrowhead=diamond]`
  • Aggregation: `edge [arrowhead=odiamond]`
- Add edge labels for relationship names and cardinality where applicable.
- Use `rankdir=TB` for class diagrams.
- Use colors: main classes in `#e3f2fd` (light blue) with `#1976d2` border, interfaces in `#d4edda` (light green) with `#28a745` border.

REFERENCE — Class Diagram:
digraph G {
    bgcolor="transparent";
    fontname="Arial";
    rankdir=TB;
    node [shape=record, fontname="Arial", fontsize=11, style=filled, fillcolor="#e3f2fd", color="#1976d2"];
    edge [fontname="Arial", fontsize=9, color="#555555"];

    User [label="{User|userId : int\lname : String\lemail : String\l|login()\llogout()\l}"];
    Order [label="{Order|orderId : int\ldate : Date\ltotal : double\l|placeOrder()\lcancelOrder()\l}"];

    User -> Order [label="places", arrowhead=open, taillabel="1", headlabel="*"];
}
